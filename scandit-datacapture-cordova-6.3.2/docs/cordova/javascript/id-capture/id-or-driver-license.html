Id cards and Driver license#

Overview#

The Scandit Id Capture supports scanning and parsing various ID card and driver’s license formats.

Id cards/US Driving License#

mrzResult

Represents data extracted from MRZ (Machine Readable Zone) of Machine Readable Travel Document (MRTD).

Dictionary with key/value pairs. See table below

driverLicenseDetailedInfo

The driver license detailed info.

Dictionary with these key/value pairs:

  • ‘restrictions’ : string. The restrictions to driving privileges for the driver license holder.

  • ‘endorsements’: string. The additional privileges granted to the driver license holder.

  • ‘vehicleClass’ : string. The type of vehicle the driver license holder has privilege to drive.

conditions

The driver license conditions.

String. Can be null.

with YYYY (0 if date is not recognized or missing), MM (in range 1 (January)-12 (December), 0 if date is partial and month is not recognized or missing), DD (day of the month. 0 if date is partial and day is not recognized or missing) as integers. Entries are omitted from the dictionary if they are not specified by the input.

Data Element ID

Meaning

Parsed Content

documentType

MRTD document type of recognized document.

String.

primaryID

The primary identifier.

String.

secondaryID

The secondary identifier.

String.

issuer

Three-letter or two-letter code which indicate the issuing State. Three-letter codes are based on Aplha-3 codes for entities specified in ISO 3166-1 with extensions for certain States. Two-letter codes are based on Aplha-2 codes for entities specified in ISO 3166-1 with extensions for certain States.

String.

dateOfBirth

Holder’s date of birth.

String.

documentNumber

Document number. Document number contains up to 9 characters. Element does not exist on US Green Card.

String.

nationality

Nationality of the holder represented by a three-letter or two-letter code. Three-letter codes are based on Alpha-3 codes for entities specified in ISO 3166-1 with extensions for certain States. Two-letter codes are based on Aplha-2 codes for entities specified in ISO 3166-1 with extensions for certain States.

String.

gender

Gender of the card holder. Gender is specified by use of the single initial capital letter F for female, M for male or < for unspecified.

String.

documentCode

Document code. Document code contains two characters. For MRTD the first character shall be A, C or I. The second character shall be discretion of the issuing State or organization except that V shall not be used, and C shall not be used after A except in the crew member certificate. On machine-readable passports (MRP) first character shall be P to designate an MRP. One additional letter may be used, at the discretion of the issuing State or organization, to designate a particular MRP. If the second character position is not used for this purpose, it shall be filled by the filter character <.

String.

dateOfExpiry

Date of expiry.

String.

opt1

First optional data. Returns empty string if not available. Element does not exist on US Green Card.

String.

opt2

Second optional data. Returns empty string if not available. Element does not exist on Passports and Visas.

String.

alienNumber

Alien number. Returns empty string if not available. Exists only on US Green Cards.

String.

applicationReceiptNumber

Application receipt number. Returns empty string if not available. Exists only on US Green Cards.

String.

immigrantCaseNumber

Immigrant case number. Returns empty string if not available. Exists only on US Green Cards.

String.

mrzText

The entire Machine Readable Zone text from ID. This text is usually used for parsing other elements. NOTE: This string is available only if OCR result was parsed successfully.

String.

isParsed

True if Machine Readable Zone has been parsed, false otherwise.

Bool.

isVerified

True if all check digits inside MRZ are correct, false otherwise.

Bool.

sanitizedOpt1

opt1 field, but without additional ‘<’ characters if they exist.

String.

sanitizedOpt2

opt2 field, but without additional ‘<’ characters if they exist.

String.

sanitizedNationality

Nationality, but without additional ‘<’ characters if they exist.

String.

sanitizedIssuer

Issuer, but without additional ‘<’ characters if they exist.

String.
